Frequently Asked Questions

General

What is Healthchecks?

Healthchecks is a monitoring tool for cron, Heroku Scheduler, or any periodic process.
Healthchecks will notify you when your scheduled tasks don't run so you can investigate before it becomes a problem.

Why would I need this?

A stitch in time saves nine.
Have you ever had a backup fail or an email campaign not go out?
Healthchecks will tell you right away when something doesn't happen so you can fix it.

What is a check?

A check looks like this: 

    $ curl https://hc-anansi-staging.herokuapp.com/ping/a3161eb8-368d-480b-aa22-1952dd85cac6.

It's a unique URL we create for you to add to jobs you want monitored.

The response message is a simple 200 and string OK

Cron already sends me output. How is this different?

Receiving an email when cron is working can be helpful, but it's hard to tell when things don't run.
The lack of an email in your inbox is hard to detect.
Healthchecks solves this problem by sending you an email if your task hasn't checked in when it should.
A second email notification will be sent once your task begins reporting again.

Which programming language can I use Healthchecks with?

Any language that can fetch a URL.

Who are you?

healthchecks.io is an open source project. The code is on GitHub,
it has BSD license, and is accepting contributions.
